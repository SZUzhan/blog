    # HTML常见问题

    1. src和href的区别？

      ​ 首先src是source的缩写，指向外部资源的位置，指向的内容会嵌入标签所在的位置，当浏览器在解析到该元素时，会暂停其他资源的下载和处理，知道该资源加载编译执行完毕，再进行其他的资源处理，因此JS脚本要放在底部而不是头部；

      ​ href是hypertext reference的缩写，指向资源所在位置，简历链接，浏览器会识别，并且并行下载资源，不会停止原来的资源处理，因此用link方式来加载CSS比起@import的方式要好，因为@import是html加载后才引用。举例，采用import引用，会先显示无样式的页面，然后再把样式放进去。

    2. HTML的理解

      ​ 语义化就是根据内容的结构化，选择合适的标签；

      ​ 语义化的优点：

      ​ 1.对机器友好，更适合搜索引擎的爬虫，有利于SEO（search engine optimization），此外，语义化支持读屏软件，可以根据文章生成目录；

      ​ 2.对开发者友好，语义化标签增加了代码的可读性，结构更加清晰，便于开发团队的开发与维护。

    3. DOCTYPE的作用

      ​ 是HTML5的一种标准通用标记语言的文档类型声明，告诉浏览器应该用什么文档类型定义来解析文档，不同渲染模式会影响浏览器对CSS和JS的解析，必须放在文档的第一行。

      ​ 浏览器渲染页面的两种模式:

      ​ 1.标准模式，在标准模式下，浏览器按照HTML与CSS标准对文档进行解析和渲染；

      ​ 2.怪异模式，在怪异模式下，浏览器则按照旧有的非标准的实现方式对文档进行解析和渲染。

      ​ 标准模式盒模型为W3C标准盒模型，怪异模式使用IE盒模型；

      ​ IE下标准模式为：content+padding+border+margin。

      ​ 怪异模式下为：content+margin(padding,border包含在content宽高中)

    4. script标签中defer和async的区别

      ​ 省流：两者都是异步执行的，不会阻塞浏览器的页面解析

      ​ defer：指外部js文件和当前html页面同时加载（异步加载），但只在当前页面解析完成之后执行js代码，存在多个 defer script 标签，浏览器（IE9及以下除外）会保证它们按照在 HTML 中出现的顺序执行，不会破坏 JS 脚本之间的依赖关系。 ​ async：指外部js文件和当前html页面同时加载（异步加载），在当前js文件加载完成后，执行js代码，多个 async 的时候，它们之间的执行顺序也不确定，完全依赖于网络传输结果，谁先到执行谁。

    5. 常用的meta标签

      ​ charset:编码类型

      ​ keywords:页面关键词

      ​ description：页面描述

      ​ refresh：页面重定向和刷新

      ​ viewpoint：视口的大小和比例

    7. img的srcset属性

      ​ 根据屏幕密度设置不同的图片，

      <img src="image-128.png" srcset="image-256.png 2x" />

    8. 行内元素和块级元素有哪些，空元素（viod）

      • 行内元素：

        <a>,<b>,<span>,<img>,<input>,<select>,<strong>

      • 块级元素：

        <div>,<ul>,<ol>,<li>,<dl>,<dt>,<dd>,<h1-h6>,<p>

      • 空元素：没有闭合标签

        br,hr,img,input,link,meta

    # CSS常见问题

    1. CSS优先级

      ​ 网页中引入多个CSS文件时，浏览器将按照以下优先级进行解析：

      1. 内联样式 （例如：style=”color: red”） 2. ID选择器 （例如：#id） 3. 类选择器 （例如：.class） 4. 标签选择器 （例如：p） 5. 通用选择器 （*） 6. 属性选择器 （例如：[type=radio]） 7. 伪类选择器 （例如：:hover） 8. 嵌套选择器 （例如：ul li） 在不同CSS文件之间，解析的优先级是一样的。但是，如果在同一个CSS文件内，通过导入式语法引入了其他CSS文件，那么被导入的CSS文件会具有更高的优先级。这是因为，在同一个CSS文件内，后定义的规则会覆盖前面的规则，而被导入的CSS文件定义在最后面，因此它们的优先级更高。

      ​ 下面是一个示例：假设在同一个CSS文件内，我们定义了一个.hello类，其中color属性为红色，然后在导入的CSS文件中，我们定义了.hello类的color属性为绿色。那么在网页中使用.hello类时，会显示绿色的字体。这是因为被导入的CSS文件的规则比前面的规则优先级更高。
